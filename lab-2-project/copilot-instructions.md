# Політика використання AI інструментів у проекті

## 1. Загальні принципи

Цей проект дотримується етичних та професійних стандартів при використанні AI-інструментів (GitHub Copilot, ChatGPT та подібних) на всіх етапах життєвого циклу програмного забезпечення.

### 1.1 Допустиме використання

- **Допомога у написанні коду**: Використання AI для генерації бойлерплейту, рутинних функцій та реалізацій стандартних паттернів
- **Рефакторинг та оптимізація**: Покращення існуючого коду, підвищення продуктивності
- **Документування**: Генерація коментарів, docstring'ів та документації
- **Тестування**: Допомога у написанні юніт-тестів та тестових сценаріїв
- **Дебагінг**: Аналіз помилок та пошук потенційних проблем
- **Освіта та навчання**: Розуміння концепцій та вивчення нових технологій

### 1.2 Заборонені практики

- **Плагіат**: Прямі копіювання чужого коду без атрибуції та адаптації
- **Вимкнення критичного мислення**: Сліпе прийняття AI-генерованого коду без аналізу
- **Порушення авторських прав**: Використання коду в порушення ліцензій
- **Маскування автоматизації**: Видання AI-генерованого коду як повністю авторського без обов'язків розкриття
- **Опущення тестування**: Розгортання непротестованого AI-генерованого коду

## 2. Вимоги до коду

### 2.1 Обов'язкові маркери

Весь AI-генерований або AI-оптимізований код **повинен містити коментар**:

```java
// AI-assisted code: [інструмент] - [призначення]
// Приклади:
// GitHub Copilot (v1.x) - генерація getter/setter методів
// ChatGPT - оптимізація алгоритму сортування
```

### 2.2 Якість коду

- Весь AI-генерований код повинен проходити code review
- AI-код має відповідати проектним рекомендаціям та конвенціям
- Необхідна перевірка на:
  - Відповідність додаткам безпеки
  - Оптимальність та ефективність
  - Читабельність та супроводжованість
  - Покриття юніт-тестами (мінімум 80%)

### 2.3 Комітити та git history

При комітуванні AI-генерованого коду використовувати префікс:
```
[AI-ASSISTED] Коротко про зміни

Деталі:
- Інструмент: GitHub Copilot
- Версія: 1.x
- Галузь: [назва функції/модуля]
- Тип: [генерація|рефакторинг|тестування]
```

### 2.4 Перевірка мовних норм

Весь код, коментарі та документація повинні відповідати нормам сучасної **української мови**. Необхідна перевірка на:

#### Русизми та запозичення

- ✗ "Загальный" → ✓ "Загальний"
- ✗ "Мусить" → ✓ "Повинен"
- ✗ "Итератор" → ✓ "Ітератор"
- ✗ "можливість обслуговування" → ✓ "можливість підтримки"
- ✗ "на увазі" → ✓ "мати на увазі" (або переписати)

#### Орфографія та пунктуація

- Перевірка орфографії всіх слів
- Правильне використання діакритичних знаків (ь, ї, є, і та ін.)
- Правильна розстановка розділових знаків

#### Граматика та стиль

- Узгодження роду, числа та виду дієслів
- Правильне використання часів
- Логічність висловлювань
- Уникнення словесних штампів та канцеляризмів

#### Приклади частих помилок

| Категорія | Прикладна помилка | Виправлення |
|---|---|---|
| Прикметники | "основной алгоритм" | "основний алгоритм" |
| Дієслова | "код мусить працювати" | "код повинен працювати" |
| Фразеологізми | "мати на увазі" | переписати речення |

#### Процес перевірки

1. **Автоматична перевірка**: Використовувати інструменти для перевірки орфографії
2. **Ручна перевірка**: Прочитати текст критично, шукаючи русизми
3. **Код-ревью**: Звернути особливу увагу на коментарі та документацію
4. **Коригування**: Виправити помилки перед комітуванням

## 3. Вимоги до звіту лабораторної

### 3.1 Розділ про AI-інструменти

Окремий розділ звіту **обов'язково** повинен містити:

#### 3.1.1 Резюме використання AI
- Список використаних AI-інструментів
- Потенційні/можливі сфери застосування
- Загальний обсяг AI-допомоги у проекті (%)

#### 3.1.2 Деталізована інформація

Для кожного використаного AI-інструменту описати:
- **Назва та версія**: GitHub Copilot, ChatGPT-4, Claude та ін.
- **Мета використання**: конкретні завдання та проблеми
- **Результат**: Як AI допоміг чи не допоміг, що було поліпшено або сповільнено
- **Перевірка якості**: Як результат був потестований та верифікований
- **Критична оцінка**: Що було змінено самостійно, які проблеми виявлені в AI-коді

#### 3.1.3 Приклади

Надати 2-3 конкретних приклади:
```
Приклад 1: Генерація unit тестів
Інструмент: GitHub Copilot
Код: [фрагмент]
Модифікації: [опис змін]
Результат: [покращення покриття з х% до y%]
```

#### 3.1.4 Висновки щодо використання AI асистента

- **Здобуті знання**: Які знання вдалося здобути при роботі з AI асистентом?
- **Покращені практики**: Які практики кодування вдалося покращити? Надав AI асистент приклади best practices в написанні коду?
- **Виявлені обмеження**: З якими обмеженнями та недоліками зіткнулися при використанні AI асистента? Чи можливо їх уникнути?

### 3.2 Значки у звіті

Рекомендується додати таблицю до звіту:

| AI-інструмент | Обсяг коду | Галузі | Ефективність |
|---|---|---|---|
| GitHub Copilot | 30% | Рефакторинг, тестування | Висока |
| - | - | - | - |

## 4. Етичні вимоги та прозорість

### 4.1 Аккаунтабельність

- Чітко зазначити, в якій частині проекту був залучений AI
- Усі члени команди повинні знати про використання AI-інструментів
- Code review має враховувати AI-генерований код із особливою увагою

### 4.2 Критичне розуміння та відповідальність

- AI-код не розглядається як замінник навчанню та розвитку
- Розробники повинні розуміти весь код, навіть AI-генерований
- Автоматизація не скасовує необхідність критичного аналізу та професійної відповідальності

### 4.3 Дотримання норм

- При публікації проекту на GitHub, мусить бути згадка про використання AI
- Всі ліцензійні питання повинні розглядатися
- При розповсюдженні коду, необхідно вказати на AI-залучені частини

## 5. Рекомендовані практики

1. **Ітеративний розвиток**: Використовувати AI як інтерактивного помічника, а не генератор фінального коду
2. **Парне програмування**: При роботі з AI, обговорити результати з колегою
3. **Документація**: Ведіть лог про яку було використано допомогу AI у вашому проекті
4. **Навчання**: Вивчайте згенерований код, щоб розуміти паттерни та підходи
5. **Атрибуція**: Завжди вказуйте на AI-допомогу у коді та документації

## 6. Інструмент для код-ревью

При review AI-генерованого коду звертати особливу увагу на:

- ✓ Безпека та відсутність вразливостей
- ✓ Продуктивність та оптимальність
- ✓ Читабельність та документування
- ✓ Тестування та покриття
- ✓ Відповідність проектним конвенціям
- ✓ Корректність логіки та алгоритмів
- ✓ Можливість підтримки та розширення

## Автоматична перевірка

Проект має налаштовані GitHub Actions для автоматичної перевірки:

- **Перевірка кодування UTF-8** - усі текстові файли повинні бути кодовані в UTF-8
- **Перевірка українського тексту** - автоматична перевірка на російські слова та фрази, що часто помилково вживаються в українських текстах

Workflow файл: [`.github/workflows/text-check.yml`](.../workflows/text-check.yml)
